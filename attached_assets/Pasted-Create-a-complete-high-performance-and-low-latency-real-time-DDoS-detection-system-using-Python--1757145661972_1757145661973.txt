Create a complete, *high-performance, and low-latency* real-time DDoS detection system using Python and Streamlit. The system must be designed to run on Replit and use my two pre-trained machine learning models to analyze live network traffic with minimal delay.

*### Core Functionality:*

1.  *Packet Capture:* Use the scapy library to capture live network packets from a user-selected interface efficiently.
2.  *Flow Assembly:* Group captured packets into network flows based on the 5-tuple (source IP, source port, destination IP, destination port, protocol). A flow should timeout and be processed after a period of inactivity (e.g., 10 seconds) to ensure timely analysis.
3.  *Feature Extraction:* For each completed flow, extract a comprehensive set of *72* numerical features. This process must be highly optimized for speed. The features must be extracted and ordered exactly as specified in the "Input Data Specification" section below.
4.  *Real-Time Prediction:* Feed the extracted features into my two pre-trained models (LucidCNN and the Autoencoder) to get a prediction. Use a hybrid logic: if either model flags the flow as an attack, it should be considered a threat.
5.  *Streamlit Dashboard:* Display all information in a clean, professional, and auto-refreshing Streamlit dashboard that presents the real-time data without lag.

*### Model & Data Specifications (VERY IMPORTANT):*

*Model 1: LucidCNN (Primary Classifier)*
* *Model File:* lucid.h5
* *Scaler File:* lucid.pkl
* *Framework:* TensorFlow/Keras.
* *Purpose:* A binary classifier that predicts if a flow is an 'Attack' or 'Benign'.
* *Preprocessing:* The input features *must* be scaled using the StandardScaler loaded from lucid.pkl.

*Model 2: Autoencoder (Anomaly Detector)*
* *Model File:* auto.pth
* *Scaler File:* auto.pkl (this file contains both the scaler and the anomaly threshold).
* *Framework:* PyTorch.
* *Purpose:* Detects anomalies. It was trained only on benign traffic. Any flow with a high reconstruction error is considered an anomaly/attack.
* *Preprocessing:* The input features *must* be scaled using the MinMaxScaler loaded from auto.pkl.
